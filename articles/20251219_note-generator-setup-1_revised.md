## はじめに：書くことは、もう一人ではできない

「書く」とは、思考を外に出して、  
もう一度自分に返してもらう行為だ。  

それをAIが助けるようになった今、

**「Noteジェネレーター」**の作り方を解説していく。

**思考の過程をGitHubに記録・蓄積し、発酵させる**。

## 🧩 システム全体の構造

Noteジェネレーターは、以下の4層で動作する。

```
┌────────────────────────────┐
│  ChatGPT (GPTs Builder)  ←─┐ 記事生成・構成
│      ↓ GitHub API連携       │
├────────────────────────────┤
│   GitHub Repository (NoteGenerator) │ 記事保存・履歴管理
├────────────────────────────┤
│   PowerShell / Node.js環境   │ ローカル保存・push処理
├────────────────────────────┤
│   React UI (PaperNoteStudio) │ 編集・プレビュー・生成UI
└────────────────────────────┘
```

AIが文章を生成し、GitHubに保存。  
その履歴が「発酵記録」として積み上がる。

## 📘 設計思想：「生成」ではなく「発酵」

文章を「書く」という行為は、  
データの入力ではなく、思考の熟成である。  

AIはその発酵過程を促す“温度管理装置”のような存在だ。  
GitHubのコミット履歴は、まるで樽の底に沈んだ酵母の記録。  

> 書くことは、発酵である。  
> そしてその香りをデジタルで保存するのが、Noteジェネレーターの使命だ。
